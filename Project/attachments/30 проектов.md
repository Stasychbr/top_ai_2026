# Единая логика развития индивидуального проекта

Каждый студент получает свою предметную область (вариант) и развивает **один и тот же проект** на протяжении всех 15 лабораторных работ — по той же схеме, что описана в тематическом плане РПД. Ниже — общий каркас, одинаковый для всех вариантов; в таблице раздела 3 указана специфика каждого варианта.

| Модуль / Лабы | Что добавляется в проект |
| --- | --- |
| Модуль 1 (Лабы 1–3) | Лаба 1: базовый инкапсулированный класс главной сущности. Лаба 2: конструкторы (по умолчанию/с параметрами/копирующий), статический счётчик экземпляров, фабричный метод. Лаба 3: иерархия наследования (абстрактный класс + 2–3 подкласса) и один общий интерфейс. |
| Модуль 2 (Лабы 4–6) | Лаба 4: хранение объектов в коллекциях (List/Map/Set) + операции `Stream API` (фильтрация, сортировка, агрегация). Лаба 5: `generic`-репозиторий/контейнер `<T>` с типобезопасными операциями. Лаба 6: собственная иерархия исключений предметной области + логирование + `JUnit`-тесты. |
| Модуль 3 (Лабы 7–8) | Лаба 7: устойчивый к ошибкам `CSV`-парсер выгрузки предметной области. Лаба 8: сериализация/десериализация в `JSON` через Jackson + один метод для `XML`. |
| Модуль 4 (Лабы 9–10) | Лаба 9: синхронизация общего изменяемого состояния проекта (защита от гонки). Лаба 10: параллельная обработка коллекции объектов через `ExecutorService`/`Future`. |
| Модуль 5 (Лабы 11–12) | Лаба 11: `DAO` и `CRUD`-операции в `PostgreSQL` для сущностей проекта. Лаба 12: транзакции, `batch insert`, `UPSERT`, пул соединений `HikariCP`. |
| Модуль 6 (Лабы 13–14) | Лаба 13: `Extract` (из источника лаб 7–8) + `Transform` (валидация и нормализация). Лаба 14: `Load` в БД идемпотентно + `DLQ` + метрики + интеграционные тесты. |
| Модуль 7 (Лаба 15) | Структурированный промпт и проверенная цепочка промптов для генерации тестов/документации своего проекта + code review сгенерированного кода. |
| Мини-проект (16) | Финальная интеграция всех частей проекта в единый воспроизводимый пайплайн (сборка, `CLI`-запуск, отчёт). |

## Как использовать

- Вариант закрепляется за студентом на первой лабораторной работе и не меняется до конца курса (сквозной номер варианта — как и предусмотрено в РПД, раздел 9 «Единый номер предметного варианта»).
- Конкретные условия и рубрики для каждой из 15 лабораторных работ формулируются под общий каркас (часть 2) с подстановкой специфики варианта (часть 3) — так задание остаётся индивидуальным, но проверяемый результат одинаков по структуре для всех студентов.
