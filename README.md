# Java-разработка для систем искусственного интеллекта: ООП и обработка данных

> Рабочая программа дисциплины, обновлённая в логике компетентностно-ролевой модели: Java, ETL, базы данных, многопоточность и применение LLM-инструментов.

## О дисциплине

Курс сохраняет фундаментальное ядро объектно-ориентированного программирования, но дополняет его темами, необходимыми для разработки прикладных решений в области искусственного интеллекта:

- промышленная разработка на Java и JVM-стеке;
- коллекции, generics, исключения и файловый ввод-вывод;
- многопоточность и конкурентная обработка данных;
- интеграция с реляционными базами данных через JDBC;
- проектирование простых ETL-пайплайнов;
- промпт-инжиниринг и критическая проверка кода, сгенерированного LLM.

## Актуальность

ООП остаётся основой промышленной разработки, однако само по себе уже не покрывает весь набор задач, возникающих при создании современных ИИ-систем. Разработчику требуется уметь работать с данными, базами данных, конкурентным выполнением, интеграционными пайплайнами и ИИ-ассистентами.

Обновлённая дисциплина связывает классическое ООП с реальными инженерными задачами: от проектирования объектной модели до создания работающего ETL-приложения с загрузкой данных в БД и документированным применением LLM.

## Цель

Сформировать у студентов системные компетенции в области промышленной Java-разработки для создания надёжного, масштабируемого и поддерживаемого серверного программного обеспечения и ETL-решений с осознанным применением ИИ-инструментов.

## Задачи

1. Освоить классы, инкапсуляцию, наследование, полиморфизм и интерфейсы.
2. Научиться использовать коллекции, generics, исключения, файловый ввод-вывод и прикладные библиотеки Java.
3. Сформировать базовые навыки многопоточного программирования и работы с реляционными БД через JDBC.
4. Реализовать простой ETL-пайплайн с обработкой ошибок, логированием, метриками и идемпотентной загрузкой.
5. Научиться применять LLM для генерации, рефакторинга и проверки кода без потери инженерного контроля.

## Компетенции и индикаторы КРМ

### PL-2 Java/Scala/Kotlin

**Компетенция:** способен применять JVM-совместимые языки программирования для решения задач в области ИИ.

- **PL-2.1.** Разрабатывает и отлаживает прикладные решения разного уровня сложности и для широкого круга конечных пользователей с использованием JVM-совместимых языков программирования, тестирует, испытывает и оценивает качество таких решений.
- **PL-2.2.** Разрабатывает и поддерживает системы обработки больших данных различной степени сложности.

На базовом уровне студент:

- применяет основные библиотеки для рутинных задач серверного программирования: ввод-вывод, простейшие примитивы многопоточности, интеграцию с базами данных;
- разрабатывает и поддерживает простые ETL-алгоритмы в пайплайнах обработки данных.

### LLM-5 Промпт-инженерия

**Компетенция:** организует взаимодействие с генеративными моделями через проектирование, анализ и применение промптов.

- **LLM-5.1.** Использует базовые шаблоны промптов.
- **LLM-5.2.** Встраивает промпты в пайплайн взаимодействия.

На базовом уровне студент:

- знает концепцию промпта и умеет составлять и структурировать промпт;
- использует простые последовательности промптов.

## Планируемые результаты обучения

По завершении дисциплины студент способен:

- проектировать объектно-ориентированные модели предметной области и реализовывать их на Java;
- использовать коллекции, generics, исключения, файловый ввод-вывод и библиотеки обработки структурированных данных;
- реализовывать конкурентную обработку данных с помощью базовых примитивов многопоточности;
- подключаться к реляционной БД через JDBC, выполнять CRUD-операции, транзакции, batch insert и UPSERT;
- проектировать и реализовывать простой ETL-пайплайн с логированием, метриками, обработкой ошибок и идемпотентностью;
- составлять структурированные промпты и цепочки промптов, проводить code review кода, сгенерированного LLM.

### Командный проект

Командный проект представляет собой Java-приложение для загрузки и очистки прайс-листов. Проект должен включать:

- поддержку CSV и JSON;
- очистку, нормализацию и обогащение данных;
- загрузку результатов в БД;
- конкурентную обработку нескольких файлов;
- обработку ошибок и логирование;
- README и историю разработки в Git;
- документированные промпты LLM;
- защиту архитектурных решений.
