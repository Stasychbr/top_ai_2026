# Фонд оценочных средств по дисциплине «Java-разработка для систем искусственного интеллекта: ООП и обработка данных»

> **Статус:** в разработке. Подготовлены индивидуальные лабораторные работы №1–15, варианты заданий и рубрики оценивания. Условия мини-проекта и экзамена будут доработаны отдельно.

Этот репозиторий предназначен для открытого и воспроизводимого представления фонда оценочных средств (ФОС) по дисциплине «Java-разработка для систем искусственного интеллекта: ООП и обработка данных». Здесь объединены рабочая программа дисциплины, модель измерения результатов обучения, контрольно-измерительные материалы (КИМ), критерии оценивания, методические рекомендации и используемые образовательные ресурсы. Результаты обучения связываются с компетентностно-ролевой моделью в области искусственного интеллекта (КРМ версии 3.0).

## 1. О дисциплине

### Цель

Сформировать у студентов системные компетенции в области промышленной Java-разработки для создания надёжного, масштабируемого и поддерживаемого серверного программного обеспечения и ETL-решений с осознанным применением ИИ-инструментов.

### Основные задачи

- освоить классы, инкапсуляцию, наследование, полиморфизм и интерфейсы;
- научиться использовать коллекции, generics, исключения, файловый ввод-вывод и прикладные библиотеки Java;
- сформировать базовые навыки многопоточного программирования и работы с реляционными БД через JDBC;
- реализовать простой ETL-пайплайн с обработкой ошибок, логированием, метриками и идемпотентной загрузкой;
- научиться составлять структурированные промпты и связанные последовательности промптов, применять ИИ-агента в заданных границах и критически проверять полученный результат.

### Пререквизиты

- основы программирования: переменные, типы, условия, циклы, функции и массивы;
- алгоритмы и структуры данных;
- дискретная математика;
- основы реляционных баз данных, SQL DDL/DML, транзакций, ACID, индексов и нормализации.

### Планируемые результаты обучения

По завершении дисциплины обучающийся сможет:

- проектировать объектно-ориентированные модели предметной области и реализовывать их на Java;
- использовать коллекции, generics, исключения, файловый ввод-вывод и библиотеки обработки структурированных данных;
- реализовывать конкурентную обработку данных с помощью базовых примитивов многопоточности;
- подключаться к PostgreSQL через JDBC, выполнять CRUD-операции, транзакции, batch insert и UPSERT;
- проектировать и реализовывать простой ETL-пайплайн с логированием, метриками, обработкой ошибок и идемпотентностью;
- составлять и проверять структурированные промпты и цепочки промптов;
- анализировать код, тесты и документацию, подготовленные ИИ-агентом, и самостоятельно принимать окончательные инженерные решения.

### Компетенции КРМ

Все компетенции и индикаторы реализуются на базовом уровне.

- **PL-2.1:** разрабатывает и отлаживает прикладные решения с использованием JVM-совместимых языков, тестирует и оценивает их качество.
- **PL-2.2:** разрабатывает и поддерживает простые ETL-алгоритмы в пайплайнах обработки данных.
- **LLM-5.1:** использует базовые шаблоны промптов, умеет составлять и структурировать промпт.
- **LLM-5.2:** использует простые связанные последовательности промптов, в которых результат одного этапа применяется на следующем.

## 4. Организация КИМ и вариантов

Каждый студенческий КИМ содержит:

- ссылку на вариант или правило продолжения сквозной предметной области;
- подробное задание без готового решения;
- границы самостоятельной реализации и использования ИИ-агента;
- перечень материалов, которые требуется представить.

Для лабораторных №1–14 подготовлено по 20 предметных вариантов. Номер варианта назначается один раз и сохраняется на протяжении всего практикума. В лабораторной №15 студент использует ту же предметную область без отдельного набора вариантов.

Критерии оценивания не дублируются в студенческих заданиях и размещены отдельно в [методических указаниях преподавателю](methodical-guidelines/teachers-assessment/README.md).

## 5. Оценивание лабораторных работ

По умолчанию каждая индивидуальная работа оценивается по 100-балльной шкале.

| Категория | Максимум |
|---|---:|
| Выполнение функциональных требований | 40 |
| Архитектура и качество реализации | 30 |
| Ошибки и граничные случаи | 15 |
| Инженерные и ИИ-инструменты | 10 |
| Производительность, когда она предусмотрена заданием | 5 |

Перевод результата:

- 90–100 — отлично;
- 75–89 — хорошо;
- 60–74 — удовлетворительно;
- менее 60 — неудовлетворительно.

Обязательное условие положительной оценки: студент способен объяснить представленный код, воспроизвести запуск и показать, как проверял результат ИИ-агента. Подробные критерии приведены в [рубриках лабораторных работ](methodical-guidelines/teachers-assessment/README.md).

Правила формирования итоговой оценки по дисциплине и промежуточной аттестации пока не утверждены.
