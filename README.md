# Фонд оценочных средств по дисциплине "Java-разработка для систем искусственного интеллекта: ООП и обработка данных"

> **Статус:** в разработке.

Этот репозиторий предназначен для открытого и воспроизводимого представления фонда оценочных средств (ФОС) по дисциплине «Java-разработка для систем искусственного интеллекта: ООП и обработка данных». Здесь объединены рабочая программа дисциплины, модель измерения результатов обучения, контрольно-измерительные материалы (КИМ), критерии оценивания, методические рекомендации и используемые образовательные ресурсы. Результаты обучения связываются с компетентностно-ролевой моделью в области искусственного интеллекта (КРМ версии 3.0).

## 1. О дисциплине

**Цель дисциплины.** Сформировать у студентов системные компетенции в области промышленной Java-разработки для создания надёжного, масштабируемого и поддерживаемого серверного программного обеспечения и ETL-решений с осознанным применением ИИ-инструментов.

**Основные задачи.** 
- Освоить классы, инкапсуляцию, наследование, полиморфизм и интерфейсы.
- Научиться использовать коллекции, generics, исключения, файловый ввод-вывод и прикладные библиотеки Java.
- Сформировать базовые навыки многопоточного программирования и работы с реляционными БД через JDBC.
- Реализовать простой ETL-пайплайн с обработкой ошибок, логированием, метриками и идемпотентной загрузкой.
- Научиться применять LLM для генерации, рефакторинга и проверки кода без потери инженерного контроля.

**Структура и содержание.** Дисциплина состоит из 7 модулей: 
- Основы ООП.
- Коллекции, обобщения и исключения.
- Ввод-вывод.
- Многопоточность.
- Базы данных и JDBC.
- ETL-пайплайн.
- Промпт-инжиниринг.
Обучение включает лекции / лабораторные работы / проект / самостоятельную работу и завершается экзаменом / защитой проекта.

**Пререквизиты и связи с другими дисциплинами.** 
- основы программирования;
- алгоритмы и структуры данных;
- дискретная математика;
- базы данных:реляционная модель, SQL DDL/DML, транзакции, ACID, индексы и нормализация.

**Планируемые результаты обучения.** По завершении дисциплины обучающийся сможет:
- проектировать объектно-ориентированные модели предметной области и реализовывать их на Java;
- использовать коллекции, generics, исключения, файловый ввод-вывод и библиотеки обработки структурированных данных;
- реализовывать конкурентную обработку данных с помощью базовых примитивов многопоточности;
- подключаться к реляционной БД через JDBC, выполнять CRUD-операции, транзакции, batch insert и UPSERT;
- проектировать и реализовывать простой ETL-пайплайн с логированием, метриками, обработкой ошибок и идемпотентностью;
- составлять структурированные промпты и цепочки промптов, проводить code review кода, сгенерированного LLM..

Полное описание приведено в [РПД](docs/rpd.md).

## 7. Порядок заполнения

1. Заполните РПД и сформулируйте проверяемые результаты обучения.
2. Выберите релевантные профессиональные роли и компетенции КРМ.
3. Для каждого результата выберите индикатор и требуемый уровень.
4. Заполните модель измерения.
5. Создайте КИМ и рубрики оценивания по шаблонам.
6. Согласуйте систему итогового балла.
7. Добавьте ресурсы, сведения о команде и лицензию.
