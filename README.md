# Java-разработка для систем искусственного интеллекта: ООП и обработка данных

> **Статус:** в разработке. Подготовлены 15 индивидуальных лабораторных работ, командный мини-проект №16, варианты и рубрики. Экзаменационные материалы дорабатываются отдельно.

Репозиторий содержит открытый и воспроизводимый фонд оценочных средств дисциплины. Курс обновляет классическое ООП через систематическую работу с данными, инженерные практики Java и контролируемое применение ИИ-агентов. Агент помогает с ограниченными артефактами, но студент проверяет результат и принимает окончательное инженерное решение.

## Цель и задачи

**Цель:** сформировать системные навыки Java-разработки для создания надёжного, поддерживаемого серверного ПО и ETL-решений с осознанным применением ИИ-инструментов.

Задачи дисциплины:

1. Освоить классы, инкапсуляцию, наследование, полиморфизм и интерфейсы.
2. Научиться использовать коллекции, generics, исключения, файловый ввод-вывод и прикладные библиотеки Java.
3. Сформировать базовые навыки многопоточного программирования и работы с реляционными БД через JDBC.
4. Реализовать простой ETL-пайплайн с обработкой ошибок, логированием, метриками и идемпотентной загрузкой.
5. Научиться составлять структурированные промпты и цепочки промптов, применять ИИ-агента в заданных границах и критически проверять результат.

### Компетенции КРМ

Основной профессиональный ролевой профиль дисциплины использует точные наименования из матрицы КРМ v3.0:

| Профессиональная роль КРМ v3.0 | Рекомендованные матрицей компетенции | Вклад дисциплины |
|---|---|---|
| Data Engineer (Инженер по данным) | PL-2 — С | обязательный уровень Б и отдельное задание С |
| MLOps (Специалист по эксплуатации ИИ) | PL-2 — С; LLM-5 — Б | обязательный Б по обоим направлениям и отдельное задание С по PL-2 |
| TADS (Технический аналитик данных) | PL-2 — С | обязательный уровень Б и отдельное задание С |

Дополнительное соответствие LLM-5 ролям AI Architect и ML Engineer, а также полная матрица приведены в [§6 РПД](docs/rpd.md#6-компетенции-и-профессиональный-ролевой-профиль-крм-v30). Дисциплина формирует часть компетенций ролевого профиля, но не присваивает профессиональную роль.

Обязательная траектория реализует Б. Для явного измерения трёхуровневой структуры ФОС содержит отдельные задания С/П в лабораторных №13–15 и проекте №16. Процентная оценка и достигнутый уровень фиксируются раздельно по [модели Б/С/П](methodical-guidelines/teachers-assessment/krm-level-model.md).

- **PL-2.1:** разрабатывает и отлаживает прикладные решения с использованием JVM-совместимых языков, тестирует и оценивает их качество.
- **PL-2.2:** разрабатывает и поддерживает простые ETL-алгоритмы в пайплайнах обработки данных.
- **LLM-5.1:** использует базовые шаблоны промптов, умеет составлять и структурировать промпт.
- **LLM-5.2:** использует простые связанные последовательности промптов, в которых результат одного этапа применяется на следующем.

## Рекомендуемая шкала оценивания

| Категория | Вес |
|---|---:|
| Технический результат | 35% |
| Корректность, тесты и граничные случаи | 25% |
| Качество кода | 10% |
| Работа с ИИ-агентом и проверка результата | 25% |
| Воспроизводимость и комплектность сдачи | 5% |

В индивидуальных [рубриках](methodical-guidelines/teachers-assessment/README.md) веса уточняются под цель работы. Производительность учитывается внутри технического результата или проверки только там, где её измерение предусмотрено КИМ.

Ориентировочный перевод результата: 90–100 — «отлично», 75–89 — «хорошо», 60–74 — «удовлетворительно», менее 60 — «неудовлетворительно». Локальные нормативы образовательной организации имеют приоритет.
