# Лабораторная работа №5. Generics: универсальные структуры данных

**Элемент РПД:** 2.3  
**Индикаторы:** PL-2.1, LLM-5.1

## Вариант задания

Основную сущность и предметные примеры выберите по [варианту лабораторной №5](../methodical-guidelines/teachers-resources/lab5_V.md). Номер варианта должен совпадать с предыдущими работами.

## Задание

1. Реализуйте обобщённый класс `Pair<A, B>` с двумя закрытыми полями разных типов, конструктором и методами чтения.
2. Реализуйте обобщённый статический метод `swap`, меняющий местами два элемента массива любого ссылочного типа. Метод должен проверять допустимость переданных индексов.
3. Создайте отдельную версию пары для сравнимых значений либо наложите корректное ограничение `Comparable` на требуемый параметр типа. Добавьте операции определения минимального и максимального элементов без приведения к конкретному классу.
4. Объявите интерфейс `Repository<T>` с операциями сохранения, поиска по идентификатору, удаления и получения всех объектов. Тип результата поиска и поведение при отсутствии объекта согласуйте с последующей лабораторной по исключениям.
5. Реализуйте репозиторий основной сущности своего варианта на основе `Map<Long, T>` либо другого типа ключа, прямо заданного вариантом. Реализация не должна хранить элементы как `Object` и выполнять небезопасные приведения.
6. Выполните предметные операции с generics, коллекциями и Stream API, указанные в варианте. Подготовьте примеры с двумя различными допустимыми типами, чтобы подтвердить универсальность решений.
7. Добейтесь отсутствия предупреждений о raw types и unchecked cast в собственном коде.

## Работа с ИИ-агентом

1. Полностью реализуйте `Pair`, `swap`, ограничения типов и `Repository<T>` самостоятельно.
2. Передайте ИИ-агенту готовый код и попросите провести проверку типобезопасности: найти raw types, необоснованные wildcard, небезопасные приведения и слишком слабые или слишком строгие ограничения типов.
3. Для каждого предложения укажите, принимаете вы его или отклоняете, и обоснуйте решение с учётом требований задания.
4. Генерацию интерфейса и реализации `Repository<T>` ИИ-агенту не поручайте.

## Что представить

- исходный код обобщённых классов, метода и репозитория;
- примеры использования с разными типами;
- текст запроса на review;
- таблицу принятых и отклонённых рекомендаций с обоснованиями.
