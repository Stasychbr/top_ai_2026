# Лабораторная работа №2. Конструкторы, перегрузка, `static`

**Элемент РПД:** 1.3  
**Индикаторы:** PL-2.1, LLM-5.1

## Вариант задания

Продолжайте работу с предметной областью и номером варианта из лабораторной №1. Требуемые для этой работы виды объектов и правила их создания приведены в [перечне вариантов](../methodical-guidelines/teachers-resources/lab2_V.md).

## Задание

1. Доработайте основной класс своего варианта и реализуйте не менее трёх конструкторов:
   - без параметров, устанавливающий осмысленные значения по умолчанию;
   - со всеми обязательными полями;
   - сокращённый, без идентификатора либо без одного из необязательных полей.
2. Организуйте делегирование между конструкторами через `this(...)`, чтобы общая инициализация не дублировалась.
3. Добавьте закрытое статическое поле-счётчик созданных объектов и открытый статический метод для чтения его значения. Счётчик должен изменяться при создании объекта через любой открытый конструктор.
4. Реализуйте статический фабричный метод для основной сущности. Метод принимает данные, указанные в варианте, назначает новый идентификатор и возвращает созданный объект.
5. Перегрузите `getDescription`: вариант без параметров возвращает полное описание, вариант с логическим параметром позволяет выбрать краткий или полный формат.
6. В `main` создайте объекты всеми предусмотренными способами, выведите оба формата описания и итоговое значение счётчика.
7. Проверьте отдельно делегирование конструкторов, уникальность автоматически назначенных идентификаторов и отсутствие двойного увеличения счётчика.

## Работа с ИИ-агентом

1. Конструкторы, фабричный метод, счётчик и перегрузку реализуйте самостоятельно.
2. Передайте ИИ-агенту сигнатуры готовых элементов и описание их фактического поведения.
3. Попросите подготовить Javadoc для открытых конструкторов и методов, а также короткие примеры их использования.
4. Проверьте каждый комментарий: параметры, возвращаемые значения, побочные эффекты и значения по умолчанию должны соответствовать коду. Исправьте документацию вручную.

## Что представить

- обновлённый исходный код;
- Javadoc для открытого API класса;
- примеры использования всех конструкторов и фабричного метода;
- текст промпта и перечень исправлений документации.
