# Лабораторная 14. ETL-пайплайн: Load + мониторинг (ETL завершён)

## Варианты

Общая задача: Завершить ETL-цикл, реализовав идемпотентную загрузку данных в БД через DAO, сбор метрик (record) и Dead Letter Queue (DLQ) для ошибочных записей.

1. Банковское дело (BankAccount):
Загрузка транзакций из CSV. UPSERT по external_id. Метрика «ошибки» — дубликаты референсов или отрицательный баланс после конвертации валют.
2. Автомобильный транспорт (Car):
Пакетная вставка автопарка дилера. Идемпотентность по VIN. DLQ — записи с невалидным годом выпуска (<1900).
3. Электронная коммерция (Product):
Синхронизация остатков со склада поставщика. UPSERT по SKU. Логирование количества товаров, ушедших в OutOfStock во время загрузки.
4. Социальные сети (UserProfile):
Миграция пользователей. UPSERT по email. DLQ — профили без подтверждения почты; метрика «загружено» только для верифицированных.
5. Геолокация (GeoPoint):
Запись GPS-трека устройства. Если точка с таким timestamp уже есть — обновлять точность (accuracy). Ошибки — координаты (0,0).
6. Университет (Student):
Зачисление абитуриентов. Проверка лимитов мест на факультете внутри транзакции Load. DLQ — недостаточные баллы ЕГЭ.
7. Видеоигры (VideoGame):
Обновление базы достижений (achievements). UPSERT по (gameId, achievementKey).
8. Медицина (PatientCard):
Импорт карт пациентов. Маска СНИЛС при загрузке. Идемпотентность по номеру полиса ОМС.
9. Библиотечное дело (LibraryItem):
Загрузка каталога ISBN. Автоматическое присвоение категории "Unknown", если код не найден в справочнике категорий (Enrichment на этапе Load).
10. Космонавтика (Spacecraft):
Загрузка телеметрии. Игнорирование пакетов, где напряжение батареи <3V (запись таких строк в DLQ как критический сбой датчика).
11. Умный дом (SmartDevice):
Регистрация устройств в контроллере. IP-адреса должны быть уникальны (UPSERT/UPDATE), иначе ошибка конфликта адреса.
12. Ресторанный бизнес (MenuItem):
Обновление меню. Пересчет цены блюда (цена ингредиентов × коэффициент) прямо перед SQL-запросом INSERT.
13. Музыка (MusicTrack):
Индексация фонотеки. Длительность трека должна быть >30 секунд, иначе запись идет в DLQ как «обрывок файла».
14. Управление проектами (Task):
Импорт задач из Jira. Построение дерева Parent-Child связей в БД. Ошибка — циклическая ссылка (A зависит от B, B от A).
15. Зоопарк (Animal):
Учет рациона. Проверка, что суммарный вес корма в порции не превышает вместимость кормушки вольера.
16. Фитнес (Exercise):
Агрегация тренировок. Расчет суммарного тоннажа (sets * reps * weight) на уровне SQL (этап Load) вместо Java.
17. Кибербезопасность (NetworkCredential):
Заливка логов фаервола. Группировка событий брутфорса по IP за минуту. Блокировка IP происходит триггером в БД при превышении лимита.
18. Сельское хозяйство (CropField):
Отчет о сборе урожая. Метод harvest(double hectares) должен атомарно уменьшать доступную площадь поля в таблице БД.
19. Event-менеджмент (ConcertTicket):
Выделение мест на схеме зала. Транзакция: проверить доступность сектора → забронировать места → создать заказ. Ошибка — место занято другим пользователем.
20. Экология (RecyclableWaste):
Прием вторсырья. Расчет эко-баллов на основе веса и типа пластика. Материал «Опасные отходы» сразу перенаправляется в таблицу исключений (DLQ).
