# Лабораторная работа №10: Асинхронное выполнение задач с возвратом результата

## Тема

**Пул потоков. Асинхронное выполнение задач с возвратом результата.**

## Задачи

1. Задача: загрузить данные о книгах из 5 разных CSV-файлов параллельно.
2. Создать ExecutorService с фиксированным пулом из 4 потоков.
3. Каждый файл загружается в отдельном Callable<List<Book>>. Результаты собираются через Future.get().
4. После загрузки всех файлов объединить списки, удалить дубликаты по ISBN.
5. Сравнить время последовательной и параллельной загрузки для 5 файлов по ~10 000 строк.
6. Реализовать таймаут для Future.get() — если файл загружается дольше 30 секунд, отменять задачу и логировать ошибку.
7. Сгенерировать обёртку с помощью ИИ для параллельной загрузки файлов — проанализировать, обработал ли он исключения и таймауты.

## Стек технологий

- **Язык:** Java (версия 17+)
- **IDE:** JetBrains IntelliJ IDEA Community Edition
- **VCS:** Git
- **ИИ-помощник:** GitHub Copilot (или аналоги)

## Инструкция по выполнению

### 1. Подготовка

Cоздайте новую ветку:

```bash
git checkout -b feature/lab10-executor
```

Подготовить 5 CSV-файлов (books_1.csv ... books_5.csv) по ~10 000 строк в каждом. Включить дубликаты ISBN между файлами.

### 2. Задача (Callable)

- Реализовать класс CsvBookLoader implements Callable<List<Book>>, принимающий путь к файлу.
- Внутри реализовать логику парсинга (можно использовать код из Лабораторной работы №7).

### 3. Параллельное выполнение (ExecutorService)

- Создать пул: Executors.newFixedThreadPool(4).
- Отправить 5 задач на выполнение через executor.submit(new CsvBookLoader("fileX.csv")). Сохранить возвращаемые объекты Future<List<Book>> в список.

### 4.Сбор результатов и обработка таймаута

- Итерироваться по списку Future.
- Вызывать future.get(30, TimeUnit.SECONDS).
- Обработать TimeoutException: вызвать future.cancel(true) и залогировать ошибку.
- Собрать все полученные списки книг в один общий List<Book>.

### 5. Дедупликация

- Объединить списки и удалить дубликаты по ISBN (используя Stream.distinct() или Set).

### 6. Бенчмарк

- Написать метод последовательного чтения тех же файлов в одном потоке.
- Сравнить время выполнения параллельной версии (п.3–5) и последовательной.

### 8. Генерация через ИИ (Copilot)

- Дать Copilot промпт: «Create a Java method that uses ExecutorService to read multiple files in parallel using Callable and returns a combined list of Books with timeout handling».
- Проверить: добавил ли ИИ блокировку прерывания (InterruptedException) внутри call()? Использует ли он правильный Collector для объединения списков?

### 8. Работа с системой контроля версий

- Зафиксируйте изменения:

```bash
git status
git add .
git commit -m "feat(lab10): implemented parallel CSV loading with ExecutorService and Future"
```

## Ожидаемый результат

Ссылка на репозиторий. Программа выводит:

- Время последовательной загрузки
- Время параллельной загрузки T/4 (или меньше, если упираемся в дисковую подсистему).
- Сообщение об ошибке таймаута, если один из файлов намеренно «заморожен» или слишком велик.
- Корректное количество уникальных книг после слияния всех 5 файлов.

## Контрольные вопросы

1. Почему при использовании пула из 4 потоков для 5 задач они не выполняются мгновенно одновременно?2. В чем разница между shutdown() и shutdownNow()?
Что произойдет, если внутри call() возникнет исключение, а мы забудем обернуть future.get() в try-catch (ExecutionException)?
3. Как поведет себя future.get(timeout), если основной поток был прерван (thread.interrupt()) во время ожидания?

## Критерии оценки

Критерии оценки: корректное использование ExecutorService с фиксированным пулом из 4 потоков для запуска пяти задач типа Callable<List<Book>>, успешный сбор результатов через Future.get() с обязательным перехватом исключений (ExecutionException) и реализацией таймаута в 30 секунд (TimeoutException) с последующей отменой задачи, объединение всех списков книг с удалением дубликатов по ISBN (требующее наличия equals/hashCode), а также наличие бенчмарка, наглядно демонстрирующего разницу во времени выполнения между последовательным чтением файлов и параллельным запуском в пуле потоков, при этом код должен быть зафиксирован отдельным коммитом с осмысленным сообщением.

## Варианты

1. Банковское дело (BankAccount): Параллельная конвертация транзакций из разных валют в RUB через Callable<Double>. Итог: сверка суммы всех сконвертированных пополнений с финальным балансом.
2. Транспорт (Car): Проверка VIN-номеров на уникальность в пуле потоков. Итог: поиск дубликатов, которые пропустила несинхронизированная проверка Set.contains().
3. E-commerce (Product): Обработка пачки заказов параллельно. Итог: подсчет разницы между «проданным» количеством и реальным остатком на складе (должна быть >0> 0>0 без синхронизации).
4. Соцсети (UserProfile): Массовая регистрация аккаунтов. Итог: создание ConcurrentHashMap.newKeySet() для логинов и сравнение скорости с обычным HashSet.
5. Геолокация (GeoPoint): Вычисление расстояний от базовой точки до списка координат в отдельных задачах. Итог: агрегация результатов поиска ближайшей точки.
6. Университет (Student): Пересчет GPA для всего потока студентов параллельно по группам. Итог: объединение отсортированных списков студентов.
7. Видеоигры (VideoGame): Установка игр на диск. Итог: предотвращение установки, если availableSpace станет отрицательным (используя атомарные операции или синхронизацию внутри Callable).
8. Медицина (PatientCard): Фоновый расчет индекса массы тела (BMI). Итог: использование Future.cancel(true) для прерывания расчета, если пользователь закрыл вкладку.
9. Библиотека (LibraryItem): Продление книг читателями одновременно. Итог: корректный учет лимита в 5 книг через AtomicInteger.
10. Космонавтика (Spacecraft): Распределение энергии между системами корабля задачами. Итог: суммарный расход не должен превышать мощность генератора.
11. Умный дом (SmartDevice): Сканирование сети на наличие устройств. Итог: заполнение карты IP-адресов без коллизий при использовании ConcurrentHashMap.
12. Ресторан (MenuItem): Приготовление блюд из одного заказа разными поварами. Итог: чек выдается только когда все Future<Готовности> завершены (invokeAll).
13. Музыка (MusicTrack): Генерация превью-роликов для загруженных треков. Итог: прогресс-бар обновляется по мере завершения Future каждой задачи.
14. Управление проектами (Task): Автоматический аудит статуса задач. Итог: список заблокированных задач собирается из результатов нескольких парсеров логов.
15. Зоопарк (Animal): Раздача корма вольерам. Итог: исключение ситуации, когда один вольер получил две порции, а другой — ноль (атомарный забор из общего склада).
16. Фитнес (Exercise): Расчет тоннажа за месяц по дням недели параллельно. Итог: суммирование результатов Future<Integer>.
17. Кибербезопасность (NetworkCredential): Анализ лог-файлов брутфорса. Итог: топ-3 атакующих IP адреса вычисляются быстрее за счет параллельного чтения кусков файла.
18. Агро (CropField): Сбор урожая с разных участков поля. Итог: общий вес продукции равен сумме результатов всех комбайнов.
19. Event-менеджмент (ConcertTicket): Рассылка электронных билетов покупателям. Итог: отслеживание количества успешно отправленных писем через результаты Future<Boolean>.
20. Экология (RecyclableWaste): Сортировка мусора на конвейере. Итог: процент брака (ушедшего не в тот бак) снижается за счет более быстрой классификации объектов потоками.
