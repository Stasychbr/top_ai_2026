# Лабораторная работа №9: Многопоточность: Runnable, Thread, synchronized

## Тема

**Создание потоков. Проблема гонки данных. Синхронизация.**

## Задачи

1. Есть общий склад книг LibraryStock с полем availableCopies и методами borrowBook() и returnBook().
2. Запустить 10 потоков-читателей, каждый выполняет 1000 операций: случайную выдачу книги или её возврат. Логика не должна позволять выдавать книгу, если копий нет.
3. Реализовать без синхронизации — продемонстрировать некорректный итоговый баланс (race condition).
4. Добавить synchronized — убедиться, что баланс сходится.
5. Сравнить производительность: замерить время выполнения synchronised-версии.
6. Реализовать альтернативу через ReentrantLock и сравнить с synchronized по времени и читаемости.
7. Сделать запрос к ИИ объяснить причину расхождения баланса в несинхронизированной версии — оценить качество объяснения.

### 2. Сущность (LibraryStock)

- Класс, представляющий склад конкретной книги.
- Поле private int availableCopies.
- Методы borrowBook() (выдача — декремент, если >0) и returnBook() (прием — инкремент).

### 3. Задача без синхронизации (Race Condition)

- Создать класс Reader implements Runnable. В конструкторе он получает ссылку на один общий объект LibraryStock.
- В методе run() выполнить цикл из 1000 итераций: случайным образом вызвать либо borrowBook(), либо returnBook().
- Запустить массив из 10 потоков (Thread), работающих с одним и тем же экземпляром склада одной книги.
- После завершения всех потоков (join()) вывести итоговое количество книг. Оно будет некорректным (например, отрицательным), так как два потока одновременно прочитали число «1», решили взять книгу и записали «0» или «-1».

### 4.Синхронизация (synchronized)

- Добавить ключевое слово synchronized перед методами borrowBook и returnBook.
- Повторить запуск. Итоговое количество копий должно вернуться к исходному значению (баланс сойдется).

### 5. Бенчмарк (synchronized)

- Замерить время выполнения цикла выдачи/возврата в защищенной версии через System.nanoTime()

### 6. Альтернатива (ReentrantLock)

- Заменить synchronized на явный замок ReentrantLock.
- Использовать конструкцию try { lock.lock(); ... } finally { lock.unlock(); } внутри методов.
- Сравнить время выполнения с версией synchronized.

## Варианты

Общая задача для всех: Есть общий ресурс (счет, склад и т.д.) с методами изменения состояния. Запустить 10+ потоков без синхронизации (показать Race Condition), затем добавить synchronized/Lock (исправить).

1. Банковское дело (BankAccount)
Ресурс: Баланс в USD.
Операции: Потоки конвертируют случайные суммы из EUR в RUB по курсу и делают deposit.
Ошибка: Из-за гонки при конвертации (курс 
× сумма) итоговый баланс в RUB неверный.
2. Транспорт (Car)
Ресурс: VIN-номер как ключ в реестре.
Операции: Потоки пытаются зарегистрировать авто с одинаковыми последними символами VIN.
Ошибка: Два потока одновременно проверяют отсутствие дубля и записывают данные, создавая дубликат.
3. E-commerce (Product)
Ресурс: Количество на складе (stock).
Операции: Потоки-клиенты одновременно заказывают последний товар.
Ошибка: Остаток становится −1, так как оба прочитали «1», решили купить и вычли единицу.
4. Соцсети (UserProfile)
Ресурс: Множество занятых логинов (Set<String>).
Операции: Двое выбирают одинаковый никнейм.
Ошибка: Оба видят, что имя свободно, и успешно регистрируются (дубликат ключа).
5. Геолокация (GeoPoint)
Ресурс: Текущие координаты объекта.
Операции: Один поток читает широту, другой в этот момент её меняет.
Ошибка: Вычисляется расстояние между точкой со старой долготой и новой широтой («скачок» трека).
6. Университет (Student)
Ресурс: Средний балл (GPA).
Операции: Добавление оценки «A» (4.0) двум студентам параллельно.
Ошибка: При пересчете среднего балла (sum + grade) / count теряются обновления одной из оценок.
7. Видеоигры (VideoGame)
Ресурс: Свободное место на диске (availableSpaceGb).
Операции: Установка двух игр общим объемом больше, чем есть места.
Ошибка: Доступное место становится отрицательным (
<0).
8. Медицина (PatientCard)
Ресурс: Поле age.
Операции: Фоновый поток обновляет возраст каждую секунду, основной печатает его.
Ошибка: На экране мелькают невалидные значения или происходит Integer overflow из-за частичного чтения long-значения времени.
9. Библиотека (LibraryItem)
Ресурс: Лимит книг у читателя (счетчик 
0…5).
Операции: Читатель берет две книги одновременно разными руками (потоками).
Ошибка: Лимит превышен (6>5), так как проверка лимита не атомарна.
10. Космонавтика (Spacecraft)
Ресурс: Уровень топлива.
Операции: Двигатель и система связи потребляют топливо одновременно.
Ошибка: Топливо уходит в минус, корабль летит с пустым баком.
11. Умный дом (SmartDevice)
Ресурс: IP-адрес устройства.
Операции: Два новых девайса подключаются к сети.
Ошибка: Контроллер выдает обоим один и тот же свободный IP (коллизия адресов).
12. Ресторан (MenuItem)
Ресурс: Остатки ингредиента (например, лосось).
Операции: Два повара готовят сложные роллы одновременно.
Ошибка: Ингредиент заканчивается посередине нарезки у обоих, блюдо испорчено.
13. Музыка (MusicTrack)
Ресурс: Счетчик прослушиваний (playCount).
Операции: Тысячи пользователей нажимают «Play» одновременно.
Ошибка: Счетчик растет очень медленно, реальные цифры не совпадают со статистикой.
14. Проекты (Task)
Ресурс: Статус задачи (status).
Операции: Менеджер закрывает задачу, а разработчик в это же время пишет коммит.
Ошибка: Коммит привязывается к уже закрытой задаче (нарушение workflow).
15. Зоопарк (Animal)
Ресурс: Порция еды в кормушке.
Операции: Две особи одного вида едят из одной миски.
Ошибка: Одна съела порцию второй, вторая осталась голодной (хотя общая сумма съеденного верна).
16. Фитнес (Exercise)
Ресурс: Вес штанги.
Операции: Два атлета меняют блины на одном грифе.
Ошибка: Общий вес снаряда оказывается непредсказуемым (то слишком тяжелым, то легким).
17. Кибербезопасность (NetworkCredential)
Ресурс: Счетчик неудачных попыток входа.
Операции: Брутфорс-атака с 10 IP адресов.
Ошибка: Блокировка аккаунта (lock account) не срабатывает вовремя, так как счетчик сбрасывается или считается некорректно.
18. Агро (CropField)
Ресурс: Норма полива участка.
Операции: Две дождевальные машины работают над границей участков.
Ошибка: Участок затоплен (превышена норма в 2 раза), соседний засох.
19. Event (ConcertTicket)
Ресурс: Место в зале (ряд+сиденье).
Операции: Два человека одновременно покупают последний билет через мобильное приложение.
Ошибка: Продано два билета на одно кресло (овербукинг).
20. Экология (RecyclableWaste)
Ресурс: Цвет мусорного бака (тип отходов).
Операции: Мусоровоз забирает пластик и стекло из совмещенного бака.
Ошибка: Из-за гонки при классификации партии (датчик сработал дважды неверно) весь груз едет на свалку вместо переработки.
