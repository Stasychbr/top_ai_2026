# Лабораторная 13. ETL-пайплайн: Extract + Transform (данные из файлов в БД)

## Тема

**Проектирование и реализация ETL-процесса. Этапы Extract и Transform.**

## Задачи

1. Extract: реализовать универсальный ридер DataSourceReader, способный читать данные из CSV и JSON (использовать паттерн «Стратегия» или конфигурацию). Источник: books.csv и books.json с идентичным набором полей.
2. Transform:
- Нормализация: привести author к формату «Фамилия И.О.» (входные данные могут быть в разнобой: «Толстой Л.Н.», «Л.Н. Толстой», «Толстой Лев Николаевич»).
- Фильтрация: исключить книги без ISBN.
- Обогащение: добавить поле category на основе ключевых слов в названии (справочник категорий хранится в отдельном CSV).
3. Результат этапа Transform — List<Book> с очищенными и обогащёнными данными.
4. Используя цепочку промптов: 1) спроектировать DataSourceReader с поддержкой CSV и JSON; 2) отдельным промптом написать метод нормализации имени автора с примерами входных и ожидаемых выходных данных.

## Инструкция по выполнению

### 1. Подготовка

Cоздайте новую ветку:

```bash
git checkout -b feature/lab13-etl
```

Подготовить файлы в src/main/resources:
- books.csv (поля: title, author, year, isbn)
- books.json (массив объектов с теми же полями)
- categories.csv (поля: keyword, category_name)

### 2. Extract (Извлечение) — Универсальный Reader

- Реализовать интерфейс DataSourceReader<T> с методом List<T> read().
- Реализовать стратегию для CSV (CsvBookReader) и JSON (JsonBookReader). Для JSON использовать Jackson.
- Создать фабрику или конфигуратор DataSourceReaderFactory, который по расширению файла (.csv / .json) возвращает нужную реализацию стратегии.

### 3. Transform (Преобразование) — Stream Processing

- Написать сервис BookProcessingService, принимающий сырой список книг.
- Нормализация автора: Реализовать метод normalizeAuthor(String rawName). Должен обрабатывать варианты "Л.Н. Толстой", "Толстой Л.Н.", "Лев Николаевич Толстой" и приводить к формату "Толстой Л.Н.".
- Фильтрация: Удалить объекты, где ISBN пустой или null.
- Обогащение (Enrichment): Загрузить справочник категорий из categories.csv в Map<String, String> (keyword → name). Добавить полю category значение, если ключевое слово найдено в title.

### 4.Результат

- Метод runPipeline() должен вернуть готовый List<Book>.

## Критерии оценки

Критерии оценки: корректная реализация паттерна «Стратегия» через интерфейс DataSourceReader и его реализации для CSV (OpenCSV/вручную) и JSON (Jackson), наличие фабрики, выбирающей ридер по расширению файла, успешное применение Stream API для цепочки трансформаций, где реализован метод нормализации имени автора с использованием регулярных выражений для обработки разных форматов ввода, фильтрация записей без ISBN, обогащение данных категориями из справочника через предварительно загруженную Map (Lookup) без вложенных циклов, а также фиксация кода в Git коммитом с осмысленным сообщением и предоставление файлов конфигурации или примеров входных/выходных данных для проверки работы нормализатора.
