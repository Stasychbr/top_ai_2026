# Рубрика лабораторной работы №1

## Проверяемые результаты

- Проект использует JDK 17+ и воспроизводимо запускается.
- Класс и поля соответствуют назначенному варианту; все поля закрыты.
- Для каждого поля реализованы корректные методы чтения и записи.
- `getDescription()` формирует устойчивое предметное описание, а `main` демонстрирует не менее трёх объектов.
- `.gitignore` исключает файлы IDE и результаты сборки.
- README, созданный с ИИ-агентом, проверен и соответствует фактическому проекту; код класса выполнен студентом.

## Существенные недостатки

- подмена варианта классом из общего примера;
- открытые поля или отсутствие обязательных методов доступа;
- README содержит неработающие команды;
- программная часть сгенерирована агентом вопреки условию.
