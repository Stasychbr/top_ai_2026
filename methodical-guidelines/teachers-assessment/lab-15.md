# Рубрика лабораторной работы №15

## Проверяемые результаты

- Одиночный промпт связан с назначенной предметной областью и содержит роль, контекст, задачу, формат, ограничения, примеры или критерии приёмки.
- Результат компилируется и проверен на сериализацию, типы, изменяемость и соответствие ограничениям.
- Три промпта образуют реальную цепочку: архитектура используется при создании модели, а проверенная модель — при создании клиента.
- Студент явно фиксирует изменения между этапами и не подменяет цепочку тремя независимыми запросами.
- Code review охватывает таймауты, HTTP-ошибки, повреждённые ответы, прерывания, секреты, логирование и зависимости.
- Итоговые исправления обоснованы и подтверждены компиляцией или тестом.

## Существенные недостатки

- сохранено общее задание `Order`, не связанное с вариантом;
- промпт не содержит ограничений и проверяемого формата;
- последующие запросы не используют результат предыдущих;
- результат агента принят без компиляции и code review;
- в материалах опубликованы секреты или закрытые данные.
