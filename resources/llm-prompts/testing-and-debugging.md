# Тестирование и отладка

Сначала запросите test plan без кода. Передайте требования, инварианты, сигнатуры и известные риски; потребуйте обычные, граничные и ошибочные сценарии. Проверьте план самостоятельно.

Только после утверждения плана передайте точные signatures, dependencies и test framework и запросите JUnit либо integration tests. Для stress test дополнительно задайте duration limit, workload, invariant и правила завершения потоков.

Запустите тесты локально. Следующим промптом передайте точное сообщение компилятора, stack trace, failed assertion и минимальный релевантный фрагмент. Просите объяснить причину и предложить локальное изменение, но не разрешайте автоматически переписывать проект. Студент решает, ошибочен тест или реализация, вносит правку и повторяет запуск.
