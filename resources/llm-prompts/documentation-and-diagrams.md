# Документация и диаграммы

Используйте инструкцию только после завершения и запуска кода.

## README

Передайте фактическое назначение проекта, Java version, build tool и проверенную команду запуска. Запросите только `README.md`: prerequisites, build, run и пример результата. Запретите придумывать зависимости и функции. Сопоставьте каждую команду с проектом.

## Javadoc

Передайте точные public signatures и фактическое поведение. Попросите описать parameters, return value, thrown exceptions, side effects и defaults. Сверьте каждый `@param`, `@return` и `@throws` с кодом.

## Mermaid или PlantUML

Передайте объявления классов, поля, public methods и связи. Укажите требуемый текстовый формат и запрет на изменение архитектуры. Проверьте inheritance, interface implementation, multiplicity, access modifiers и совпадение имён.

Техническую часть промпта пишите на английском; русское описание предметной области допустимо. Сохраните исходный ответ и список ручных исправлений.
