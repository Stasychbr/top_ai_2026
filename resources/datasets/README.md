# Учебные датасеты для мини-проекта №16

В каталоге находятся стартовые CSV-наборы для командного
[мини-проекта «Серверный ETL-сервис»](../../Project/README.md). Они позволяют
начать работу с грязными данными, но не заменяют подготовку полного входного
набора по требованиям КИМ.

Команда выбирает один вариант из вариантов своих участников. Готовые CSV
предоставлены только там, где исходная предметная область набора достаточно
близка одному из 20 сквозных вариантов курса. Поэтому номер в имени файла —
номер исходного набора, а не номер варианта курса; используйте таблицу ниже.

## Формат

Каждый файл содержит строку заголовка и 200 строк данных:

- кодировка UTF-8 с BOM (`utf-8-sig`);
- разделитель — запятая;
- поля со встроенными запятыми заключены в кавычки по правилам CSV;
- 18 столбцов с общей логикой, но доменными названиями.

| № | Группа столбцов | Назначение |
|---:|---|---|
| 1–2 | `ID`, `Код записи` | технический идентификатор и устойчивый бизнес-ключ |
| 3–4 | основное наименование и его повтор | сверка дублирующих значений и выбор канонического |
| 5–7 | связанная персона, категория и подкатегория | доменные связи и справочники |
| 8–11 | город, адрес, телефон и email | очистка строк и проверка форматов |
| 12 | доменный статус | нормализация по справочнику |
| 13–14 | количественный и стоимостной показатели | приведение и проверка чисел |
| 15–16 | даты создания и обновления | разбор дат и выбор актуальной версии |
| 17–18 | система-источник и примечания | происхождение записи и необязательный текст |

Конкретные заголовки приведены в первой строке каждого файла. Перед
реализацией команда составляет таблицу соответствия «исходный столбец →
Java-поле → столбец БД → правило проверки и очистки».

## Какие проблемы заложены в данные

Наборы содержат несколько видов намеренно неоднородных значений:

- корректные сокращения городов, категорий и статусов, которые требуется
  раскрыть по справочнику;
- мусорные и неизвестные текстовые значения;
- некорректные числа, даты, телефоны и адреса email;
- пустые и обрезанные значения;
- расхождения между основным и дублирующим столбцами.

Точное распределение команда определяет самостоятельно и сопоставляет со
своими правилами валидации. Готовая статистика и эталон очистки не
предоставляются.

## Как использовать

Предоставленный CSV — стартовый fixture для знакомства со схемой и
регрессионной проверки. Обязательный набор мини-проекта должен включать не
менее 1000 записей в минимум трёх файлах, причём должны присутствовать оба
формата: CSV и JSON. Простое копирование строк не считается расширением набора.

Рекомендуемый порядок:

1. проверить число и типы столбцов, определить бизнес-ключ;
2. измерить фактические классы проблем и решить, какие записи очищать, а какие
   отправлять в DLQ;
3. дополнить fixture либо заменить его собственным набором;
4. проверить объём, разнообразие, распределение ошибок и межфайловые дубликаты;
5. сохранить способ подготовки данных и ожидаемую сводку вместе с проектом.

Для подготовки собственного набора используйте
[инструкцию по генерации данных](../llm-prompts/data-generation.md). За
проверенный генератор и воспроизводимый прогон на 10 000–100 000 строк
предусмотрены дополнительные баллы согласно
[рубрике проекта](../../methodical-guidelines/teachers-assessment/project.md).
