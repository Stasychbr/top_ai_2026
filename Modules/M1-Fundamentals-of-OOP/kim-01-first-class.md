# Лабораторная работа №1. Настройка окружения. Первый класс

**Элемент РПД:** 1.2  
**Индикаторы:** PL-2.1, LLM-5.1

## Вариант задания

Используйте номер варианта, назначенный преподавателем. Предметная область, имя класса и обязательные поля приведены в [перечне вариантов](../../methodical-guidelines/teachers-resources/lab1_V.md). Во всех последующих лабораторных работах сохраняйте тот же номер варианта.

## Порядок выполнения

Перед началом изучите [инструкцию по промптам](../../resources/llm-prompts/documentation-and-diagrams.md). Технические формулировки, API, signatures и diagnostics передавайте на английском.

1. **Самостоятельно.** Установите JDK 17 или новее и IntelliJ IDEA Community Edition. Проверьте версию JDK из командной строки и в настройках проекта.
2. **Самостоятельно.** Создайте Java-проект и подключите его к Git. Добавьте `.gitignore`, исключающий как минимум `.idea`, `*.iml`, `out`, `target` и `*.class`.
3. **Самостоятельно.** Создайте класс основной сущности своего варианта. Объявите все указанные в варианте поля закрытыми (`private`) и выберите для них подходящие типы Java.
4. **Самостоятельно.** Для каждого поля реализуйте открытый метод чтения. Имя метода должно начинаться с `get`, а для логического поля допускается `is`.
5. **Самостоятельно.** Для каждого изменяемого поля реализуйте открытый метод записи с одним параметром и возвращаемым типом `void`. Имя метода должно начинаться с `set`.
6. **Самостоятельно.** Добавьте метод `getDescription()`, который возвращает одну строку с основными сведениями об объекте. Формат строки определите применительно к предметной области и используйте одинаково для всех объектов.
7. **Самостоятельно.** Создайте класс с методом `main`. Создайте не менее трёх объектов с различающимися данными, задайте их состояние через методы записи и выведите результат `getDescription()` для каждого объекта.
8. **Проверка результата.** Запустите программу и убедитесь, что все поля представлены в описании корректно, а служебные файлы IDE не попали в репозиторий.

9. **Проверка результата.** Сначала самостоятельно завершите и проверьте программную часть.
10. **С ИИ-агентом.** Передайте ИИ-агенту название проекта, назначение реализованного класса, используемую версию Java и фактический способ запуска.
11. **С ИИ-агентом.** Попросите подготовить черновик `README.md` с кратким описанием проекта, требованиями к окружению и инструкцией по запуску.
12. **Проверка результата.** Сопоставьте инструкцию с проектом, исправьте неточные команды и удалите неподтверждённые утверждения. Код класса и метода `main` ИИ-агенту не поручайте.
