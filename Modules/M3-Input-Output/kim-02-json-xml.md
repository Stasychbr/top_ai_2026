# Лабораторная работа №8. Файловый ввод-вывод: JSON и XML

**Элемент РПД:** 3.3  
**Индикаторы:** PL-2.1, LLM-5.1

## Вариант задания

Формат предметных данных, дополнительные поля и операции преобразования приведены в [варианте лабораторной №8](../../methodical-guidelines/teachers-resources/lab8_V.md). Номер варианта сохраняется.

## Порядок выполнения

Перед началом изучите [инструкцию по промптам](../../resources/llm-prompts/code-generation-and-review.md). Технические формулировки, API, signatures и diagnostics передавайте на английском.

1. **Самостоятельно.** Подключите Jackson Databind и Jackson Dataformat XML через Maven или Gradle. Зафиксируйте версии зависимостей.
2. **Самостоятельно.** Подготовьте JSON-файл с массивом объектов своей сущности. Состав полей должен соответствовать модели предыдущих работ и варианту.
3. **Самостоятельно.** Настройте POJO или record для сериализации. Примените `@JsonProperty` при различии имён, `@JsonIgnore` для поля, которое не должно попадать в файл, и `@JsonFormat` для даты с явно заданным шаблоном.
4. **Самостоятельно.** Самостоятельно реализуйте чтение JSON в типизированную коллекцию и запись коллекции обратно в JSON. Не используйте необобщённые `Map` вместо предметной модели.
5. **Самостоятельно.** Самостоятельно реализуйте конвертер CSV → JSON на основе результата лабораторной №7. Некорректные CSV-записи обрабатывайте по ранее принятому правилу.
6. **Самостоятельно.** Определите сигнатуру одного метода чтения XML: тип входа, возвращаемый тип и поведение при ошибках. Реализацию этого метода выполните в рамках задания с ИИ-агентом.
7. **Самостоятельно.** Реализуйте запись данных в XML через `XmlMapper` самостоятельно и проверьте обратное чтение полученным методом.
8. **Самостоятельно.** Сравните исходные данные с данными после циклов JSON → Java → JSON и XML → Java → XML. Отдельно проверьте даты, специальные символы, пустые значения и неизвестные поля.

9. **Проверка результата.** Сначала самостоятельно завершите JSON-часть и конвертацию CSV → JSON.
10. **С ИИ-агентом.** Подготовьте спецификацию одного метода чтения XML: сигнатуру, структуру XML, модель данных, допустимые исключения и не менее трёх проверочных примеров.
11. **С ИИ-агентом.** Передайте спецификацию ИИ-агенту и поручите реализацию только этого метода.
12. **Проверка результата.** Встройте полученный метод, скомпилируйте проект и запустите проверки. Проверьте типизацию, закрытие ресурсов и обработку повреждённого XML.
13. **Проверка результата.** Исправьте реализацию самостоятельно и зафиксируйте, какие требования агент выполнил или нарушил.

## Что представить


- исходный код, файл сборки и модели данных;
- примеры CSV, JSON и XML;
- проверки циклического преобразования;
- спецификацию метода, промпт, ответ ИИ-агента и перечень исправлений.
- ссылку на использованный шаблон или инструкцию по промпту;
- точные тексты промптов, существенные ответы ИИ-агента и журнал внесённых исправлений.
