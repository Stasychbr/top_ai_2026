# Экзамен / зачет

Промежуточная аттестация подтверждает освоение дисциплины в целом и не должна дублировать только один модуль.

## Индекс ресурсов

| № | Ресурс | Назначение |
|---:|---|---|
| 1 | [Вопросы к экзамену](attachments/Вопросы%20к%20экзамену.md) | перечень из 47 вопросов по дисциплине |
| 2 | [Генератор билетов](attachments/Генератор%20билетов.md) | промпт-шаблон для распределения вопросов по 25 билетам |

## Генератор билетов

Генератор представляет собой проверяемый промпт-шаблон, а не отдельную программу. Его нужно передать выбранному чат-интерфейсу или coding agent вместе с файлом либо полным текстом списка вопросов.

Шаблон запрашивает 25 уникальных билетов по три вопроса:

- два вопроса из фундаментальной части №1–35;
- один вопрос из блоков ETL и ИИ-инструментов №36–47;
- включение всех 47 вопросов хотя бы в один билет;
- не более трёх использований одного вопроса;
- вывод в чистом Markdown без исходной нумерации вопросов.

### Порядок использования

1. Откройте [список вопросов](attachments/Вопросы%20к%20экзамену.md) и [промпт генератора](attachments/Генератор%20билетов.md).
2. Передайте оба материала ИИ и получите полный Markdown со всеми билетами.
3. Проверьте результат до публикации: должно быть ровно 25 заголовков билетов и 75 строк вопросов, в каждом билете — два фундаментальных и один современный вопрос.
4. Дополнительно проверьте присутствие всех 47 исходных вопросов, отсутствие полностью одинаковых билетов и частоту каждого вопроса не более трёх раз.

Результат ИИ нельзя принимать без проверки: языковая модель может пропустить вопрос, изменить его формулировку, превысить допустимое число повторов или нарушить распределение по блокам.
