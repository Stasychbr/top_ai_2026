# Лабораторная работа №11. JDBC: подключение к БД и CRUD

**Элемент РПД:** 5.2  
**Индикаторы:** PL-2.1, LLM-5.1

## Вариант задания

Имя таблицы, предметная сущность, ограничения и дополнительные запросы возьмите из [варианта лабораторной №11](../methodical-guidelines/teachers-resources/lab11_V.md). Номер варианта сохраняется.

## Задание

1. Запустите PostgreSQL 16 локально или в Docker. Параметры подключения вынесите из исходного кода в конфигурацию либо переменные окружения; пароль не фиксируйте в репозитории.
2. Подготовьте SQL-скрипт создания таблицы для сущности варианта. Добавьте первичный ключ, обязательные поля, уникальное ограничение и предметные проверки, указанные в варианте.
3. Подключите JDBC-драйвер через Maven или Gradle и реализуйте получение соединения с БД.
4. Объявите интерфейс DAO, отделяющий SQL от остального приложения. Предусмотрите операции:
   - вставка одного объекта;
   - пакетная вставка;
   - выборка всех объектов;
   - выборка по предметному критерию варианта;
   - обновление одного изменяемого поля;
   - удаление по уникальному признаку.
5. Самостоятельно спроектируйте интерфейс DAO и реализуйте не менее четырёх перечисленных операций. Ровно одну из оставшихся операций разрешается реализовать с ИИ-агентом; остальные реализуйте самостоятельно.
6. Для входных параметров используйте `PreparedStatement`. Не формируйте SQL конкатенацией пользовательских значений.
7. Закрывайте `Connection`, `PreparedStatement` и `ResultSet` через `try-with-resources`. Преобразуйте строки результата в типизированные объекты предметной модели.
8. Для вставки получите созданный идентификатор, а для обновления и удаления проверьте число изменённых строк.
9. Подготовьте интеграционные проверки всех операций, включая нарушение уникальности, отсутствие записи и специальные символы в строковых полях.

## Работа с ИИ-агентом

1. После самостоятельного проектирования DAO и реализации не менее четырёх операций выберите одну нереализованную операцию.
2. Составьте структурированный промпт: передайте интерфейс, DDL таблицы, модель, сигнатуру метода, правила обработки ошибок и критерии приёмки.
3. Поручите ИИ-агенту реализацию только выбранного метода.
4. Проверьте `PreparedStatement`, параметры SQL, `try-with-resources`, обработку `SQLException`, возвращаемое значение и отсутствие секретов в коде.
5. Запустите интеграционные тесты и исправьте реализацию самостоятельно.

## Что представить

- исходный код DAO и предметной модели;
- SQL-скрипт создания таблицы;
- конфигурацию без секретов;
- интеграционные тесты и результат запуска;
- промпт, исходный ответ ИИ-агента и перечень исправлений.
