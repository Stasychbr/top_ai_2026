# Лабораторная работа №15. Составление и структурирование промптов

**Элемент РПД:** 7.2  
**Индикаторы:** LLM-5.1, LLM-5.2

## Вариант задания

Продолжайте предметную область, назначенную преподавателем в лабораторных №1–14. Основную сущность возьмите из своего варианта №1, а сценарий внешнего API выберите применительно к той же области. Отдельного варианта для этой работы нет.

## Задание

1. Выберите основную сущность своего варианта и определите для неё не менее шести разнотипных полей: идентификатор, строковое описание, числовой показатель, статус, дата или время и составное либо коллекционное значение. Используйте уже существующую модель, если она удовлетворяет требованиям.
2. Составьте структурированный промпт для получения Java-модели этой сущности. В промпте явно задайте:
   - роль и необходимую квалификацию ИИ-агента;
   - контекст предметной области;
   - точную задачу и перечень полей;
   - версию Java и допустимую форму модели;
   - требования к Jackson-аннотациям;
   - ограничения и запрещённые зависимости;
   - требуемый формат ответа;
   - критерии, по которым ответ будет принят.
3. Получите ответ и проведите проверку: скомпилируйте код в отдельной ветке или временном пакете, проверьте типы, сериализацию, именование, изменяемость и соответствие ограничениям. Не заменяйте этим кодом существующую модель без обоснования.
4. Выберите одну операцию внешнего API, уместную для варианта: получение справочной информации, загрузка статуса, курса, каталожных данных или другой предметной информации. Зафиксируйте назначение операции, пример запроса и ожидаемого ответа.
5. Выполните связанную последовательность из трёх промптов:
   - запросите архитектуру REST-клиента, модель ответа и стратегию обработки ошибок без реализации;
   - передайте утверждённую архитектуру и запросите реализацию только модели ответа;
   - передайте архитектуру и проверенную модель, затем запросите реализацию клиента на стандартном `HttpClient`.
6. В каждом следующем промпте явно используйте результат предыдущего этапа и отмечайте внесённые студентом исправления.
7. Для всех полученных фрагментов проведите code review:
   - компиляция и соответствие Java 17;
   - таймауты и обработка сетевых ошибок;
   - обработка кодов HTTP и некорректного тела ответа;
   - отсутствие секретов и небезопасного логирования;
   - корректная работа с прерываниями;
   - именование, зависимости и соответствие архитектуре.
8. Сравните первоначальные и исправленные результаты. Для каждого существенного изменения укажите причину и способ проверки.

## Работа с ИИ-агентом

1. Во всех промптах используйте нейтральное обозначение инструмента и не опирайтесь на функции, доступные только одной конкретной модели.
2. Не передавайте пароли, токены, персональные данные и закрытый код.
3. Не принимайте результат без компиляции и проверки. Ответственность за окончательное решение и исправления несёт студент.
4. Сохраните точный текст каждого промпта, значимые ответы и связь между этапами цепочки.

## Что представить

- `README.md` с описанием предметного сценария;
- структурированный одиночный промпт и результат его проверки;
- цепочку из трёх промптов и существенные ответы;
- исправленные фрагменты в отдельном пакете или ветке;
- таблицу code review с найденными проблемами, решениями и способом проверки.
